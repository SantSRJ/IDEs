1. Introducción a los IDEs

Un IDE (Entorno Integrado de Desarrollo) es un programa que combina un editor de código, depurador, terminal y otras herramientas en un solo lugar. Es mucho mejor que escribir en Notepad porque te ayuda a escribir código sin errores.

¿Qué incluye un IDE?

  • Editor: Resaltado de colores, autocompletado
  • Depurador: Buscar errores en tu código
  • Terminal: Ejecutar comandos sin salir del programa
  • Extensiones: Agregar funcionalidades extra
  • Control de versiones: Rastrear cambios con Git

IDEs vs Editor de texto

Característica IDE Editor (VS Code, Sublime)
Autocompletado ✓ Inteligente ✓ Básico
Depuración ✓ Integrada ✗ Necesita plugins
Peso (memoria) Pesado (500 MB+) Ligero (100-200 MB)
Velocidad de inicio Lenta (5-10s) Rápida (<2s)
Costo Varía Gratuito

4. Las 10 Extensiones Más Importantes

Las extensiones son plugins que agregan funcionalidades a VS Code. Son lo que lo hace tan poderoso.

Cómo instalar una extensión

  1. En VS Code, hacer clic en el icono de "Extensions" (izquierda, parece una cuadrícula)
  2. Buscar la extensión por nombre
  3. Hacer clic en "Install"

Las 10 extensiones esenciales

1. Python (Microsoft) ⭐ IMPRESCINDIBLE

¿Para qué? Soporte completo para Python: ejecución, autocompletado, debugging.

Cómo instalar: Buscar "Python" en extensions, la primera de Microsoft, click "Install".

2. Pylance (Microsoft)

¿Para qué? Detecta errores antes de ejecutar el código.

Beneficio: Si escribes algo mal, verás una línea ondulada roja.

3. Black Formatter (Microsoft)

¿Para qué? Formatea automáticamente tu código a PEP 8 (estándar Python).

Cómo usar: Presiona Ctrl+Shift+P, escribe "Format Document", Enter.

Resultado: Tu código se ve limpio y profesional sin esfuerzo.

4. Flake8 (Microsoft)

¿Para qué? Detecta errores y problemas de estilo en tiempo real.

Cómo instalar Flake8 en tu PC:

pip install flake8

5. Pytest (Microsoft)

¿Para qué? Ejecutar pruebas unitarias de tu código.

Cómo instalar Pytest en tu PC:

pip install pytest

6. Thunder Client (Thunder Client)

¿Para qué? Probar APIs sin salir de VS Code (como Postman).

Cuándo lo necesitas: Cuando hagas backend con Flask o FastAPI.

7. GitLens (Eric Amodio)

¿Para qué? Ver el historial de Git en el editor.

Beneficio: Saber quién cambió cada línea de código y cuándo.

8. Prettier (Prettier)

¿Para qué? Formatea HTML, CSS, JSON (útil si mezclas con web).

9. Better Comments (Aaron Bond)

¿Para qué? Colorea tus comentarios según tipo (TODO, FIXME, etc.).

Ejemplo:

# TODO: Agregar validación
# FIXME: Bug en la función sumar
# HACK: Solución temporal

10. Relative Path (Jakob101)

¿Para qué? Autocompletado de rutas de archivos.

Instalar las 10 extensiones rápidamente

Copia cada nombre en el buscador de extensions (Ctrl+Shift+X):

  • Python (Microsoft)

  • Flake8 (Microsoft)

5. Proyecto Práctico: Calculadora Python

Estructura del proyecto

mi_calculadora/
├── main.py
├── calculadora.py
└── README.md

Paso 1: Crear carpeta

mkdir mi_calculadora
cd mi_calculadora
code .

Paso 2: Crear calculadora.py

Botón derecho en el explorador → New File → escribir "calculadora.py"


# calculadora.py
def sumar(a, b):
    """Suma dos números."""
    return a + b

def restar(a, b):
    """Resta dos números."""
    return a - b

def multiplicar(a, b):
    """Multiplica dos números."""
    return a * b

def dividir(a, b):
    """Divide dos números."""
    if b == 0:
        return "Error: No se puede dividir por cero"
    return a / b

Paso 3: Crear main.py


# main.py
from calculadora import sumar, restar, multiplicar, dividir

print("=" * 40)
print("   CALCULADORA SIMPLE")
print("=" * 40)

a = 10
b = 5

print(f"\nSuma: {a} + {b} = {sumar(a, b)}")
print(f"Resta: {a} - {b} = {restar(a, b)}")
print(f"Multiplicación: {a} * {b} = {multiplicar(a, b)}")
print(f"División: {a} / {b} = {dividir(a, b)}")

print("\n" + "=" * 40)

Paso 4: Ejecutar

En VS Code, presionar Ctrl+F5 o click en el botón Play (arriba a la derecha).

Salida esperada:
========================================
   CALCULADORA SIMPLE
========================================

Suma: 10 + 5 = 15
Resta: 10 - 5 = 5
Multiplicación: 10 * 5 = 50
División: 10 / 5 = 2.0

========================================

Uso de extensiones en este proyecto

  • Python: Al escribir sumar(, aparecen los parámetros automáticamente
  • Pylance: Si escribes mal un nombre de función, te lo marca en rojo
  • Black Formatter: Ctrl+Shift+P → "Format Document" → tu código se formatea automáticamente
  • Better Comments: Escribe # TODO: mejorar función → aparece en rojo

6. Introducción a Git

¿Qué es Git?

Git es un programa que rastraea cambios en tus archivos. Te permite:

  • Ver quién cambió qué y cuándo
  • Volver a versiones anteriores si algo se daña
  • Trabajar con otras personas sin sobrescribir cambios
  • Crear versiones "ramas" de tu código

¿Qué es GitHub?

GitHub es un sitio web donde puedes subir tu código con Git. Es como Google Drive, pero para código.

https://github.com

Instalar Git

Windows

  1. Ir a https://git-scm.com
  2. Descargar "Git for Windows"
  3. Ejecutar el instalador, siguiente, siguiente...
  4. Listo

Mac

brew install git

Linux

sudo apt install git

Verificar instalación

git --version

Deberías ver algo como: git version 2.40.0

7. Comandos Básicos de Git

Paso 1: Configuración inicial

Haz esto solo UNA VEZ. Git necesita saber quién eres:

git config --global user.name "Tu Nombre"
git config --global user.email "tu@email.com"

Por ejemplo:

git config --global user.name "Juan Pérez"
git config --global user.email "juan@gmail.com"

Paso 2: Inicializar Git en tu proyecto

En tu carpeta del proyecto (mi_calculadora), abre terminal y ejecuta:

git init
Salida: "Initialized empty Git repository..."

Paso 3: Ver qué cambió

git status
Salida:
Untracked files:
    calculadora.py
    main.py
    README.md

(no hay cambios agregados al commit)

Explicación: Git ve tus archivos pero no los está rastreando todavía.

Paso 4: Agregar un archivo

Para que Git comience a rastrear un archivo específico:

git add calculadora.py

Paso 5: Agregar TODOS los archivos

Para agregar todos los archivos de una vez:

git add .

(El punto significa "todos")

Tip: Después de hacer git add ., ejecuta git status de nuevo. Los archivos ahora estarán en verde (listos para guardar).

Paso 6: Guardar cambios (hacer commit)

Un "commit" es como tomar una foto de tu código en este momento:

git commit -m "Inicial: Agregar proyecto calculadora"
Salida:
[main a1b2c3d] Inicial: Agregar proyecto calculadora
3 files changed, 45 insertions(+)
create mode 100644 calculadora.py

Notas:

  • El número a1b2c3d es el ID único de tu commit (siempre diferente)
  • El texto después de -m es el mensaje (debe ser descriptivo)
  • Usa verbos: "Agregar", "Corregir", "Actualizar"

Paso 7: Hacer más cambios

Edita calculadora.py y agrega una nueva función:


def potencia(a, b):
    """Eleva a a la potencia b."""
    return a ** b

Luego:

git add calculadora.py
git commit -m "Agregar función potencia"

Paso 8: Ver el historial

Para ver todos tus commits:

git log --oneline
Salida:
b2c3d4e Agregar función potencia
a1b2c3d Inicial: Agregar proyecto calculadora

Explicación: Ves todos los cambios en orden, del más reciente al más antiguo.

Ver detalles de un commit

git log

Muestra toda la información: autor, fecha, mensaje, y qué cambió.

Ver exactamente qué cambió

git show a1b2c3d

Reemplaza a1b2c3d con el ID de tu commit.

Resumen de flujo básico

  1. git status - Ver qué cambió
  2. git add . - Agregar cambios
  3. git commit -m "mensaje" - Guardar cambios
  4. git log --oneline - Ver historial

Repite estos 4 pasos cada vez que termines de hacer algo importante.

8. Volver a una Versión Anterior

¿Qué pasa si cometes un error?

Git te permite volver a cualquier versión anterior. No hay problema.

Caso 1: No quiero guardar estos cambios (aún no hiciste commit)

Si editaste un archivo pero TODAVÍA no hiciste git commit, puedes descartar cambios:

git checkout -- calculadora.py
Advertencia: Esto elimina tus cambios locales. No lo puedes deshacer.

Caso 2: Quiero volver a una versión anterior (después de commit)

Primero, ve tu historial:

git log --oneline
Salida:
c3d4e5f Agregar función logaritmo
b2c3d4e Agregar función potencia
a1b2c3d Inicial: Agregar proyecto calculadora

Digamos que quieres volver a b2c3d4e (antes de agregar logaritmo):

git reset --soft b2c3d4e
¿Qué pasa? Vuelves al commit anterior, pero TUS CAMBIOS siguen ahí (puedes verlos con git status). Es seguro.

Caso 3: Volver atrás COMPLETAMENTE (peligroso)

Si quieres descartar TODO (incluyendo tus cambios):

git reset --hard b2c3d4e
⚠️ CUIDADO: Esto elimina permanentemente todos los cambios posteriores. No lo hagas a menos que estés seguro.

Caso 4: Ver cómo era el código antes

Sin cambiar nada, solo para mirar:

git show b2c3d4e:calculadora.py

Esto muestra el contenido del archivo en ese commit específico.

Resumen - Volver atrás

Situación Comando ¿Es seguro?
Descartar cambios locales (sin commit) git checkout -- archivo.py No - Pierdes cambios
Volver atrás (guardar cambios) git reset --soft ID Sí - Cambios siguen ahí
Volver atrás (descartar todo) git reset --hard ID No - Pierdes todo
Solo ver cómo era git show ID:archivo Sí - Solo miras

9. Ramas: Desarrollo Paralelo Básico

¿Qué es una rama?

Una rama es una "copia" de tu código donde puedes trabajar sin afectar la rama principal. Es como tener dos versiones del proyecto al mismo tiempo.

¿Por qué usarlas?

  • Trabajar en nuevas funciones sin romper el código principal
  • Varias personas pueden trabajar en paralelo
  • Experimentar sin miedo a arruinar todo

Rama principal vs rama secundaria

main (o master): La rama principal, el código que funciona y está en "producción".

feature/nueva-funcion: Rama de trabajo donde desarrollas algo nuevo.

Paso 1: Ver ramas actuales

git branch
Salida:
* main

(El asterisco * indica en qué rama estás)

Paso 2: Crear una rama nueva

git branch feature/raiz-cuadrada

Paso 3: Cambiar a esa rama

git checkout feature/raiz-cuadrada
Salida: Switched to branch 'feature/raiz-cuadrada'

O crear y cambiar en UN COMANDO:

git checkout -b feature/raiz-cuadrada

Paso 4: Trabajar en la rama

Edita calculadora.py y agrega:


import math

def raiz_cuadrada(a):
    """Calcula la raíz cuadrada."""
    if a < 0:
        return "Error: No se puede calcular raíz de número negativo"
    return math.sqrt(a)

Ahora haz commit:

git add calculadora.py
git commit -m "Agregar función raiz_cuadrada"

Paso 5: Volver a main

git checkout main

Nota: Tu código EN MAIN NO TIENE la función raiz_cuadrada. Está solo en la rama feature/.

Paso 6: Unir ramas (Merge)

Cuando terminaste de trabajar en la feature y quieres agregarla a main:

git merge feature/raiz-cuadrada
Salida:
Updating a1b2c3d..b2c3d4e
Fast-forward
calculadora.py | 7 ++++++
1 file changed, 7 insertions(+)

Explicación: Los cambios de feature/ ahora están en main. ¡Listo!

Paso 7: Eliminar rama usada

Ya no necesitas la rama feature/:

git branch -d feature/raiz-cuadrada

Diagrama simple de ramas

main: A---B---C---D (commit inicial → agregar → trabajar → merge feature) \ / feature/ B1-C1 (rama paralela) Después de merge: main: A---B---C---D---D' (contiene cambios de feature)

Buenas prácticas de nombres de ramas

Tipo Nombre Ejemplo
Nueva funcionalidad feature/... feature/agregar-busqueda
Corrección de bug bugfix/... bugfix/corregir-login
Documentación docs/... docs/actualizar-readme

Resumen de ramas

  • git branch - Ver ramas
  • git branch nombre - Crear rama
  • git checkout nombre - Cambiar a rama
  • git checkout -b nombre - Crear y cambiar
  • git merge nombre - Unir rama a donde estás
  • git branch -d nombre - Eliminar rama

10. GitHub: Subir tu código a la nube

¿Por qué usar GitHub?

  • Backup de tu código (no se pierde si se daña tu PC)
  • Compartir código con otros
  • Portafolio para mostrar a empleadores
  • Colaborar en equipo

Paso 1: Crear cuenta GitHub

  1. Ir a https://github.com
  2. Hacer clic en "Sign up"
  3. Ingresar email, contraseña, usuario
  4. Verificar email

Paso 2: Crear repositorio en GitHub

  1. En GitHub, arriba a la derecha, hacer clic en "+" → "New repository"
  2. Nombre: mi_calculadora
  3. Descripción: "Calculadora simple con Python"
  4. Seleccionar "Public"
  5. NO marcar "Initialize with README"
  6. Hacer clic en "Create repository"

Paso 3: Conectar tu computadora con GitHub

GitHub te mostrará comandos. Copia y ejecuta en tu carpeta local:

git branch -M main
git remote add origin https://github.com/TU_USUARIO/mi_calculadora.git
git push -u origin main

Reemplaza TU_USUARIO con tu nombre de usuario en GitHub.

Ejemplo real:

git branch -M main
git remote add origin https://github.com/juan_perez/mi_calculadora.git
git push -u origin main
Salida:
Enumerating objects: 3, done...
...
To https://github.com/juan_perez/mi_calculadora.git
 * [new branch]  main -> main
Branch 'main' set up to track remote branch 'main' from 'origin'.

Paso 4: Verificar en GitHub

Recarga la página en GitHub. ¡Verás tu código subido!

Paso 5: Hacer cambios locales y subirlos

En tu computadora, haz cambios:

# Editar calculadora.py
git add .
git commit -m "Mejora: agregar validación"
git push

git push sube tus cambios a GitHub.

Paso 6: Descargar cambios de GitHub

Si trabajas desde otra computadora o alguien más hace cambios:

git pull

Esto descarga los cambios de GitHub a tu computadora.

Resumen de GitHub

Acción Comando
Ver repositorios remotos git remote -v
Subir cambios a GitHub git push
Descargar cambios de GitHub git pull
Ver URL del repositorio remoto git remote -v

Flujo completo: hacer cambios y subirlos

  1. git add . - Agregar cambios
  2. git commit -m "mensaje" - Guardar localmente
  3. git push - Subir a GitHub

Y si trabajas en equipo:

  1. git pull - Descargar cambios de otros
  2. (hacer tu trabajo)
  3. git add . && git commit -m "msg" && git push - Todo en uno
